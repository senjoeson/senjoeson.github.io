如何用Idea配置最基本的web项目

  • 后台学习笔记
  • 2018-02-05

前言

一直有着学习Java后台的想法,最近到年底了,正好手头上没事做,就来学习一下web项目,由于之前只学习过一点皮毛,所以,一切都从头开始学习吧!!

在学习的过程中,因为不太熟悉整个流程的配置,因此,在开始的时候花费了很多的时间去琢磨和练习,当然获得的经验,令人满意的!!

备注:首先用IDEA的原因是因为我之前用AS进行开发,用起来比较得心应有,并且IDEA是一个非常优秀的工具,提供很多便利,我也打听了很多人在用IDEA这个工具,所以,如果还在坚持用EC或者myEC,不妨来试试这个吧!

首先我们用IDEA 创建一个Java web :

然后点击Finished即可!

拿到一个空白的项目.我们应该想到以下流程:

配置web.xml
配置tomcat服务器

那么到底是先配置tomcat服务器还是先配置web.xml呢?
经过多次的测试和操作,发现其实应该是先配置好web.xml,然后再进行配置tomcat,为什么这么说呢?我先埋个伏笔!
下面我先配置web.xml

点击添加会弹出这个对话框!拉到最底部,会发现有一个web


在项目中添加一个web.xml文件,一般选择默认的就好,所以点击OK 即可添加成功!

配置完成后的结果显示如下:

ok,我们已经完成了web.xml配置,下面我们来配置Tomcat服务器吧!


单击Edit Configuration 会弹出如下对话框!

单击加号,选中Tomcat server 然后选中local 会弹出如下对话框!(备注:有人说没有找到Tomcat选项,你可以在上图中的”31 items more (…..)”)找找.

细心的同学,发现了上图中其实报了一个错误

单击Fix按钮,会自动生成

要知道 ,我们tomcat项目配置,可以直接放一个war包,即可以进行访问,因此需要添加一个war包!
这里单击OK 了就完成Tomcat配置了!
下面我们就可以启动我们的tomcat 了!

当你看到我这样结果,就证明你已经配置Tomcat成功了!

有些人就郁闷了,你都讲完了配置,却没告诉我为什么要先配置Web.xml再配置Tomcat,这不是坑么?
老铁, 我想告诉你的是,先Tomcat配置,会出现这样的问题,

它还是会提示你缺少了War包,还是得需要去先配置web.xml,然后进行生成!
按照EC以前敲代码的原则,是先撸完代码.然后再来配置服务器,因此,先配置web.xml 可以避免很多奇怪的问题!!

下面简单的说一下最近遇到的坑:
1.关于配置servlet问题

servlet-class 里面这个值一定是带包名的全路径
url-pattern 从名字上来看,url的匹配规则, 指的是通过根目录下的login路径,
可以访问到Login.class(Login是一个servlet文件)

2.关于jsp提交表单时候的action路径

我们先要从理论角度出发来看,这样可以不用让你被迷惑住,一个表单,肯定是要提交给一个servlet进行数据处
理的!那么action 肯定要定向到servlet的,这里的路径就要填写你生成的servlet路径中,不然是无法被扫描到的!!
否则你点击提交数据的时候 ,会一直出现 server500 这个错误!!

有人说万事开头难,我真的是一直被难住在了开头,为了付出了几十个小时的代价去搞懂一些配置.不过最后的结果也是值得的!我对web项目有了一个基本的认识!

希望把我遇到的一些问题和得到的一些经验,给大家分享一下 ,能够帮助到大家,由于很有很多东西不懂,如果你发现文中有错误,或者需要补充的地方,请给我留言,大家一起学习!
